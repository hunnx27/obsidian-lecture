# 💡 자세 저장 실수 방지

## 저장 전 반드시 확인할 것

```
1. 토크 OFF 실제로 되는지
   on → run,1,1000 → 팔이 움직이는지 확인
   → 안 움직이면 initMotorCommunication() 주석 해제 했는지 확인

2. 팔을 받친 상태에서 저장하기
   한 손으로 팔을 받치고 다른 손으로 위치 잘 잡기
   → 팔을 놓는 순간 다시 연습 가능

3. 저장 후 바로 run으로 확인
   off → save → 1,INITIAL → quit
   → on → run → 1,1000
   → 팔이 기대한 위치로 가는지 눈으로 확인!
```

---

## 자주 하는 실수 정리

| 실수 | 원인 | 해결 |
|------|------|------|
| 저장했는데 팔이 안 움직임 | 토크 OFF 실패 | `initMotorCommunication()` 주석 해제 확인 |
| 자세는 맞는데 새로 저장하면 이상한 값 | EEPROM 쓰레기 값 | `eeprom_clear` 먼저 실행 |
| run 명령 말을 듣는데 안 감 | 저장된 자세 없음 | `list`로 확인 후 재저장 |
| 저장 후 위치가 다름 | 손으로 놓는 순간 빗나감 | 토크 OFF 백업 후 다시 저장 |
| GRIP_PICK 자세가 블록에 닿지 않음 | 저장 위치 미세 조정 필요 | SagMeasure.ino로 처짐량 확인 후 보정 |

## GRIP과 PUT 자세 저장 팁

```
GRIP_UPPER 저장 시:
  실제 블록을 놓고 집게가 블록에 닿는 위치에서 저장
  판만 닿는 것이 아니라 집게가 감싸는 위치에서!

PUT 자세 저장 시:
  실제 블록을 들고 배치할 위치에 세워서
  집게가 열릴 때 블록이 점수 위에 떨어지는 위치 확인
```

---

## 저장 순서 권장

```
INITIAL(1) → MISSION_1(2) → SEE_STORAGE(4)
→ GRIP_UPPER(5) → GRIP_UPPER_PICK(6)
→ GRIP_LOWER(7) → GRIP_LOWER_PICK(8)
→ PUT들(10,11,13,15)

이 순서대로 run으로 하나씩 확인하면서 저장하면
나중에 어떤 자세에서 문제인지 파악이 심플해짐
```
