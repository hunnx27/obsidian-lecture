# 🏁 Ch.6 — 당일 체크리스트

> 경연 당일 **실수없이** 준비하는 체크리스트

---

## ⏰ 경연 전날 체크

- [ ] **EEPROM 저장 완료** 확인 (`list` 명령어로 목록 출력)
- [ ] **run** 명령어로 **모든 자세 동작** 실제 확인
- [ ] Pixy2 시그니처 **4개 모두 인식** 되는지 확인
- [ ] 대회 코드로 **교체 업로드** 완료

---

## 확인 순서 표

| 순서 | 확인 내용 | 확인 방법 |
|------|-----------|-----------|
| 1 | EEPROM 자세 목록 | `list` 명령어 |
| 2 | 팔 자세 동작 | `run 1,1000` ~ `run 15,1000` |
| 3 | Pixy2 4색 감지 | 시리얼 모니터 출력 |
| 4 | SLOT_PSD 실측값 | PSD 실측 코드 |
| 5 | 대회 코드 업로드 | Arduino IDE |
| 6 | 전체 1회 시뮬레이션 | 실제 동작 전 검증 |

---

## 합 한 듯 체크할 것들

### 카메라
- [ ] 현장 조명에서 Pixy2 시그니처 재학습
- [ ] 4색 모두 교차 오염 없이 감지되는지 시리얼로 확인
- [ ] 카메라 램프 켜면 감지 더 잘 되는지 확인

### 로봇팔
- [ ] INITIAL 자세에서 팔이 안전한 위치에 있는지
- [ ] GRIP_UPPER/LOWER 집기 자세가 실제 블록에 닿는지
- [ ] PUT 자세들이 정확한 위치에 내려놓는지

### PSD
- [ ] 4개 슬롯 위치 실측값으로 SLOT_PSD[] 수정
- [ ] SP(6cm), FP(43cm), MDZ(14cm) 값도 현장에서 재확인

### 코드
- [ ] 대회 코드로 교체 업로드 완료
- [ ] 배터리 충전 상태 확인

---

## ⚠️ 당일 트러블슈팅 가이드

| 증상 | 원인 | 해결 |
|------|------|------|
| 팔이 안 움직임 | EEPROM 저장 누락 | `list`로 확인 후 재저장 |
| 블록을 못 찾음 | 카메라 Range 문제 | PixyMon에서 Range 재조정 |
| 슬롯에 안 서는지 엄한 곳에 서 | SLOT_PSD 문제 | 실측값으로 수정 |
| 집는데 놓칠 | 판 자세 문제 | GRIP_PICK 자세 재저장 |
| 색깔 오분류 | 시그니처 학습 문제 | Pixy2 재학습 |

---

## 💡 최종 점검 멘틈

> 이 세 가지가 완벽하면 경연은 돌아간다.

**1. 카메라로 순서 파악** → Pixy2 학습 + 실측
**2. 팔 자세 저장** → SavePose 코드 + run 작동 확인
**3. 거리값 보정** → SLOT_PSD[] 현장 실측값 입력
