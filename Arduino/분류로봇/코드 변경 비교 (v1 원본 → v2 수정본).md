# 🔀 코드 변경 비교 (v1 원본 → v2 수정본)

이 페이지는 **원본 코드(v1)** 와 **수정된 대회 코드(v2)** 의 차이를 모듈별로 정리한 문서야.

> 💡 두 코드는 같은 미션을 수행하지만 설계 철학이 완전히 달라.

---

## 🔄 전체 변경 요약

| 항목 | 원본 v1 | 수정본 v2 |
|------|---------|-----------|
| 이동 방식 | 거리 이동 + PSD 보정 혼합 | PSD PID 이동만 사용 |
| 블록 탐색 | 카메라 X좌표로 좌우 스캔 | 슬롯 위치 이동 후 색깔 확인 |
| 블록 처리 수 | 1개만 처리 후 종료 | 전체 블록 순차 처리 |
| 배치 위치 | 고정 (8번 칸) | 색깔별 다른 위치 |
| 복귀 | 없음 (정지) | PSD 경계 감지 후 복귀 |
| 코드 구조 | 절차형 (순서대로) | 루프 기반 (반복 처리) |

## 1. 헤더 & 전역 변수

### 변경 내용

```diff
// 원본
#include "Debug.h"
#include "Manipulator.h"
#include "Mobilebase.h"
// → Pins.h 없음

// 수정본
+ #include "Pins.h"   // SW1 버튼, LED 핀 추가
```

```diff
// 원본: 단일 블록만 처리
uint8_t targetBlockSigmap = 0x00;
uint8_t goalPosition = 8;   // 항상 8번 칸에 놓음

// 수정본: 전체 블록 배열로 확장
+ #define MAX_BLOCKS 6
+ uint8_t Pixy_List[6] = {0,};   // 블록 처리 순서 배열
+ int num_x[6] = {500,};         // X좌표 저장
+ uint8_t Pixy_B = 0;            // 감지된 블록 총 개수
+ int Put_Position_ByColor[7] = {0,13,0,0,10,15,11};
```

### 수정 이유
원본은 1개 블록만 고정 위치에 놓는 구조. 수정본은 색깔별로 다른 위치에 배치하기 위해 배열로 확장.

## 2. 자세(Pose) ID 정의

### 변경 내용

```diff
// 원본: 매직 넘버 계산 방식
enum ManipulatorPoseID {
  INITIAL_AND_MISSION_INSTRUCTION = 1,
  STORAGE,
  PRE_GRIP_UPPER_BLOCK,
  GRIP_UPPER_BLOCK,
  PRE_GRIP_LOWER_BLOCK,
  GRIP_LOWER_BLOCK
};
#define MANIPULATOR_MISSION_FULFILLMENT_POSE_START_ID 6
// 사용: MANIPULATOR_MISSION_FULFILLMENT_POSE_START_ID + goalPosition
//       → 6 + 8 = 14번 자세 (뭔지 직관적으로 모름)

// 수정본: 명확한 이름으로 전부 정의
+ enum ManipulatorPoseID {
+   INITIAL = 1, MISSION_1 = 2, SEE_STORAGE = 4,
+   GRIP_UPPER = 5, GRIP_UPPER_PICK = 6,
+   GRIP_LOWER = 7, GRIP_LOWER_PICK = 8,
+   PUT_1=9, PUT_2=10, ... PUT_8=16
+ };
```

### 수정 이유
원본의 `MANIPULATOR_MISSION_FULFILLMENT_POSE_START_ID + goalPosition`은 코드만 봐서는 몇 번 자세인지 알 수 없음. 수정본은 각 자세를 명시적 이름으로 정의해서 가독성 향상.

## 3. 이동 방식 ← 가장 큰 변경점

### 변경 내용

```diff
// 원본: 절대 거리 이동 + PSD 보정 혼합
// 모드 전환 필요
ChangeMobilebaseMode2ExtendedPositionControlWithTimeBasedProfileMode(dxl);
DriveDistanceAndMmPerSecAndDirection(dxl, 450.0);  // 450mm 직진
while(!CheckIfMobilebaseIsInPosition(dxl)) {}

DriveXYDistanceAndMmPerSec(dxl, 60.0, 350.0);     // 대각선 이동
while(!CheckIfMobilebaseIsInPosition(dxl)) {}

// 다시 모드 전환
ChangeMobilebaseMode2VelocityControlMode(dxl);
while(1) {
  GetValueFromSideLeftPSDSensor(&slPSDValue6);
  GetValueFromFrontRightPSDSensor(&frPSDValue6);
  if (!LocatingWithTwoSensors(...)) break;
}

// 수정본: PSD PID만 사용 (모드 전환 없음)
+ PID_AlignAndApproach(dxl, OBS, ...);
+ DriveLeftWithSideLeftSensorPID(dxl, MDZ, ...);
+ AlignUsingFrontAndSide430PID(dxl, FP, SP, ...);
```

### 수정 이유
원본은 거리 이동(엔코더 기반)과 PSD 보정을 번갈아 사용해 모드 전환(`ChangeMobilebaseMode2...`)이 필요하고 오차가 누적됨. 수정본은 PSD만으로 처리해서 항상 실제 위치 기준으로 이동.

## 4. 블록 스캔 방식

### 변경 내용

```diff
// 원본: 비트마스크로 특정 색만, 1개만 저장
const uint8_t BLOCK_SIG_MAP = 0b00001001;
pixy.ccc.getBlocks(true, BLOCK_SIG_MAP);
targetBlockSigmap = (1 << pixy.ccc.blocks[0].m_signature-1);  // 첫 블록만!

// 수정본: 전체 스캔 + X좌표 버블정렬
+ pixy.ccc.getBlocks();
+ Pixy_B = pixy.ccc.numBlocks;
+ // X좌표 기준 버블정렬
+ for (int i=0; i<Pixy_B-1; i++)
+   for (int j=0; j<Pixy_B-1-i; j++)
+     if (num_x[j] > num_x[j+1]) { 교환 }
+ // → Pixy_List[]에 왼→오른 순서로 저장
```

### 수정 이유
원본은 가장 크게 인식된 블록 1개만 저장. 수정본은 모든 블록을 왼→오른 순서로 정렬해서 여러 블록을 순서대로 처리.

## 5. 블록 탐색 루프

### 변경 내용

```diff
// 원본: 카메라 X좌표로 좌우 이동하며 탐색
do {
  pixy.ccc.getBlocks(true, targetBlockSigmap);
  if (pixy.ccc.numBlocks &&
      blocks[0].m_x > 126 && blocks[0].m_x < 189) {  // 화면 중앙?
    // 블록 발견 → 미세 조정 → 집기
    while(1) {
      int16_t blockXError = blocks[0].m_x - PIXY2_X_SETPOINT;
      if (!DriveWithOneSensor(dxl, blockXError, ...)) { 집기 }
    }
  }
  GetValueFromFrontRightPSDSensor(&frPSDValue7);
} while (DriveUntilNoObstacleWithOneSensor(..., RIGHT));  // 오른쪽으로 이동하며 반복

// 수정본: 미리 정해진 슬롯 위치로 이동하며 탐색
+ while (findBoxNum > 0) {
+   for (int slot=0; slot<=NUM_SLOTS; slot++) {
+     DriveLeftWithSideLeftSensorPID(dxl, SLOT_PSD[slot], ...);
+     pixy.ccc.getBlocks();
+     if (blocks[i].m_signature == targetColor) {
+       Grap_blocks(y);  // 집기
+       PUT_ON(putPos);  // 배치
+     }
+   }
+ }
```

### 수정 이유
원본은 카메라가 블록을 화면 중앙에 잡을 때까지 좌우로 이동하는 방식. 조명이나 색상 변화에 민감하고 불안정. 수정본은 미리 알고 있는 슬롯 위치로 딱 이동해서 해당 색깔 블록이 있는지만 확인. 훨씬 결정론적.

## 6. 집기 자세 구조

### 변경 내용

```diff
// 원본: 코드 두 군데에 복붙 (중복!)
// 7번 위치에서 집기
RunManipulatorPoseWithPoseDataInEEPROM(dxl, PRE_GRIP_UPPER_BLOCK, 1000, 0.0);
delay(1200);
RunManipulatorPoseWithPoseDataInEEPROM(dxl, GRIP_UPPER_BLOCK, 1000, 0.0);
delay(1500);
// ... 8번 위치에서 집기 (동일 코드 반복)

// 수정본: 함수로 캡슐화
+ void Grap_blocks(int blockY) {
+   bool isUpper = (blockY < UPPER_STANDARD);
+   if (isUpper) {
+     RunManipulatorPoseWithPoseDataInEEPROM(dxl, GRIP_UPPER, 1000);
+     RunManipulatorPoseWithPoseDataInEEPROM(dxl, GRIP_UPPER_PICK, 1000);
+     CloseGripper(pixy);
+     RunManipulatorPoseWithPoseDataInEEPROM(dxl, GRIP_UPPER, 1000);
+   } else { /* 아래층 버전 */ }
+ }
```

### 수정 이유
원본은 집기 코드가 두 군데에 복붙되어 있어서 수정 시 두 곳 다 바꿔야 함 (버그 위험). 수정본은 `Grap_blocks()` 함수로 묶어서 중복 제거.

## 7. 배치(PUT) 방식

### 변경 내용

```diff
// 원본: 항상 8번 칸 고정
RunManipulatorPoseWithPoseDataInEEPROM(dxl, STORAGE, 800, -90.0);  // 회전
RunManipulatorPoseWithPoseDataInEEPROM(dxl,
  MANIPULATOR_MISSION_FULFILLMENT_POSE_START_ID + 8, 1000);  // 항상 8번
OpenGripper(pixy);

// 수정본: 색깔별 분류 배치
+ int putPos = Put_Position_ByColor[targetColor];
+ // 빨강(sig1)→13번, 초록(sig4)→10번, 파랑(sig5)→15번, 보라(sig6)→11번
+ PUT_ON(putPos);
+
+ void PUT_ON(int putPos) {
+   RunManipulatorPoseWithPoseDataInEEPROM(dxl, putPos, 1000);
+   OpenGripper(pixy);
+   RunManipulatorPoseWithPoseDataInEEPROM(dxl, pose, 1000);  // 복귀
+ }
```

### 수정 이유
원본은 무조건 8번 칸만 사용. 경연 미션은 색깔별로 다른 위치에 배치해야 하므로 `Put_Position_ByColor[]` 배열로 매핑.

## 8. 종료 처리

### 변경 내용

```diff
// 원본: 블록 1개 처리 후 무한 루프 정지
// END
while(1) { delay(1000); }  // 사실상 그냥 멈춤

// 수정본: PSD 경계 감지 후 정상 복귀
+ // 미션수행존 → 미션지시존 경계 통과
+ while(1) { if(SL_PSD_Save() > 15) break; Backward(dxl, 250); }
+ while(1) { if(SL_PSD_Save() < 15) break; Backward(dxl, 250); }
+ while(1) { if(SL_PSD_Save() > 15) break; Backward(dxl, 250); }
+
+ RunManipulatorPoseWithPoseDataInEEPROM(dxl, INITIAL, 1000);
+ Left(dxl, 150);
+ Stop(dxl, 0);
```

### 수정 이유
원본은 미션 완료 후 그냥 정지. 경연에서는 제자리로 돌아와야 점수를 받을 수 있어서 PSD로 구역 경계를 감지하며 정상 복귀하도록 수정.
